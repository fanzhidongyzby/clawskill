## 平台简介

OpenClaw 技能平台是一个开放的技能分发和管理系统，允许开发者发布、分享和更新技能，同时为用户提供便捷的技能发现和安装体验。

### 核心功能

- **技能市场**: 浏览、搜索和发现技能
- **技能安装**: 一键安装所需技能
- **版本管理**: 支持技能版本更新和回滚
- **开发者工具**: 发布和管理自己的技能
- **管理后台**: 审核和管理用户及技能

## 快速开始

### 前置要求

- **Web 界面**: 现代浏览器（Chrome、Firefox、Safari、Edge）
- **CLI 工具**: Node.js 16+ 或通过 npm 安装

### 安装 CLI 工具

```bash
# 使用 npm 安装
npm install -g @openclaw/cli

# 或使用 yarn
yarn global add @openclaw/cli
```

### 首次登录

```bash
# 使用 CLI 登录
openclaw login
```

按照提示输入用户名和密码完成登录。

## Web 界面使用

### 访问平台

打开浏览器访问: `http://localhost:3000`

### 技能浏览

1. **搜索技能**
   - 在搜索框输入关键词
   - 支持中文和英文搜索
   - 按回车键或点击搜索图标

2. **筛选技能**
   - 使用分类下拉菜单筛选
   - 支持的分类：生产力、自动化、AI工具、开发工具、集成、实用工具

3. **排序结果**
   - 最热门：按下载量和评分排序
   - 最新：按发布时间排序
   - 下载量：按总下载量排序
   - 评分：按用户评分排序

4. **查看详情**
   - 点击技能卡片打开详情弹窗
   - 查看完整描述、版本信息、作者等
   - 点击"安装技能"按钮安装

### 技能管理

进入"技能管理"页面可以：

1. **发布新技能**
   - 点击"发布新技能"按钮
   - 填写技能名称、描述、分类、版本等信息
   - 添加相关标签
   - 提交发布

2. **编辑技能**
   - 点击技能列表中的"编辑"按钮
   - 修改技能信息
   - 保存更新

3. **删除技能**
   - 点击"删除"按钮
   - 确认删除操作

4. **版本管理**
   - 展开技能行查看版本历史
   - 查看每个版本的变更说明
   - 使用"回滚"按钮恢复到历史版本

### 管理后台

管理员可以访问后台管理功能：

1. **用户管理**
   - 查看所有注册用户
   - 编辑用户信息
   - 封禁违规用户

2. **技能审核**
   - 查看待审核技能
   - 审核通过或拒绝技能
   - 查看审核历史

3. **审核日志**
   - 查看所有操作日志
   - 追踪系统活动

## CLI 工具使用

### 基本命令

#### 搜索技能

```bash
# 基础关键词搜索
openclaw search "自动化"

# 启用语义搜索（更智能）
openclaw search "提高工作效率" --semantic

# 按分类筛选
openclaw search "AI" --category ai

# 限制结果数量
openclaw search "开发" --limit 5

# 查看更多结果
openclaw search "工具" --offset 20
```

#### 安装技能

```bash
# 安装指定技能
openclaw install <skill-id>

# 安装特定版本
openclaw install <skill-id> --version 1.2.0

# 强制重新安装
openclaw install <skill-id> --force
```

#### 列出已安装技能

```bash
# 查看已安装技能
openclaw list

# 查看所有版本
openclaw list --all
```

#### 更新技能

```bash
# 更新指定技能
openclaw update <skill-id>

# 更新到特定版本
openclaw update <skill-id> --version 2.0.0

# 更新所有技能
openclaw update
```

#### 卸载技能

```bash
# 卸载技能（需要确认）
openclaw uninstall <skill-id>

# 直接卸载（跳过确认）
openclaw uninstall <skill-id> --yes
```

### 开发者命令

#### 发布技能

```bash
# 从当前目录发布
openclaw publish

# 从指定目录发布
openclaw publish --path ./my-skill

# 指定版本号
openclaw publish --version 2.0.0

# 预发布模式（不实际发布）
openclaw publish --dry-run
```

#### 版本管理

```bash
# 查看技能版本信息
openclaw version <skill-id>

# 回滚到历史版本
openclaw rollback <skill-id>
# 然后选择目标版本
```

### 认证命令

```bash
# 登录
openclaw login

# 登出
openclaw logout

# 查看当前用户信息
openclaw whoami
```

## 技能管理

### 技能分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 生产力 | 提高工作效率的工具 | 日程管理、文档处理 |
| 自动化 | 自动化任务执行 | 批量处理、定时任务 |
| AI工具 | AI 驱动的功能 | 文本生成、图像处理 |
| 开发工具 | 开发辅助工具 | 代码生成、调试工具 |
| 集成 | 第三方服务集成 | API 调用、数据同步 |
| 实用工具 | 日常实用功能 | 格式转换、数据处理 |

### 版本号规范

遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范：

- `MAJOR.MINOR.PATCH`（主版本.次版本.修订号）
- 主版本：不兼容的 API 变更
- 次版本：向下兼容的功能性新增
- 修订号：向下兼容的问题修正

示例：
- `1.0.0` - 初始版本
- `1.1.0` - 新增功能
- `1.1.1` - bug 修复
- `2.0.0` - 重大更新（不兼容）

### 标签使用

为技能添加相关标签可以：

- 提高搜索可见性
- 帮助用户快速定位
- 展示技能特性

推荐标签：
- 功能相关：`自动化`、`AI`、`批量处理`
- 应用场景：`办公`、`开发`、`数据分析`
- 技术栈：`Python`、`JavaScript`、`API`

## 最佳实践

### 开发者指南

#### 1. 技能命名

- 使用清晰、描述性的名称
- 避免使用缩写和模糊的词汇
- 包含核心功能关键词

**好的示例:**
- "批量图片处理助手"
- "API 自动化测试工具"

**避免:**
- "Helper"
- "Tool V1"

#### 2. 描述撰写

- 清晰说明技能的功能
- 列出主要特性和优势
- 提供使用场景示例
- 包含必要的依赖说明

#### 3. 版本发布

- 遵循语义化版本规范
- 在 changelog 中记录变更
- 小步迭代，频繁发布
- 保留旧版本以支持回滚

#### 4. 文档完善

- 提供完整的 README
- 包含使用示例
- 说明配置和参数
- 更新维护状态

#### 5. 测试验证

- 发布前充分测试
- 考虑不同使用场景
- 处理异常情况
- 提供错误提示

### 用户建议

- 根据实际需求选择技能

- 阅读用户评价

- 了解技能的依赖和配置
- 优先选择活跃维护的技能

## 常见问题

### 安装相关

**Q: 安装技能时提示权限错误？**

A: 确保您已登录并且有足够的权限。尝试重新登录：

```bash
openclaw logout
openclaw login
```

**Q: 如何查看已安装技能的详细信息？**

A: 使用 version 命令：

```bash
openclaw version <skill-id>
```

**Q: 可以同时安装多个版本的同一技能吗？**

A: 不可以，每个技能只能安装一个版本。但可以随时回滚到历史版本。

### 搜索相关

**Q: 语义搜索和关键词搜索有什么区别？**

A: 
- **关键词搜索**: 精确匹配关键词，适合查找特定技能
- **语义搜索**: 理解意图，推荐相关技能，适合探索和发现

**Q: 搜索结果不准确怎么办？**

A: 
- 尝试不同的关键词
- 使用语义搜索获取更智能的结果
- 添加分类筛选
- 查看完整技能列表手动筛选

### 开发相关

**Q: 技能发布后多久会在市场上显示？**

A: 技能发布后需要经过审核，审核通过后即可在市场上显示。审核时间通常在 24 小时内。

**Q: 如何更新已发布的技能？**

A: 
1. 修改技能代码和 package.json 中的版本号
2. 使用 `openclaw publish` 命令重新发布
3. 新版本会自动推送更新给已安装用户

**Q: 可以下架已发布的技能吗？**

A: 可以。在技能管理页面或使用 CLI 工具可以下架技能。下架后新用户无法安装，但已安装用户可以继续使用。

### 管理相关

**Q: 如何成为开发者？**

A: 注册账号后在个人设置中申请开发者权限，或联系管理员。

**Q: 审核被拒绝怎么办？**

A: 查看拒绝原因，修改技能后重新提交。如有疑问可联系管理员咨询。
